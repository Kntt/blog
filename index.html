下面开始学习过程:

模版语法

1.文本

数据绑定最常用都语法之一, “Mustache”语法(双大括号)插值

对于文本数据的绑定,使用场景太多了,是最基础的用法

<template>
    <div>
        <p>hello {{world}}</p>
        <p v-text="'hello ' + world"></p>
        <p>{{`hello ${world}`}}</p>
        <p v-text="`hello ${world}`"></p>
        <button @click="world='vue2.0'">改变wrold值</button>
     </div>
</template>

<script>

export default {
     data () {
         return {
              world : "world"
         }
     }
}

</script>

上面的demo使用了四种绑定文本的语法

第一:“Mustache”语法 {{world}}

第二: v-text指令

第三和第四是对第一第二两种写法的 ES6 语法扩展;

这种语法绑定方式,会一直监听 world 的值, worl的值变化时候,页面也随之刷新;

还有下面一种绑定方法, 一次性绑定, 不会继续监听

2.v-once

该赋值指令, 只对文本进行一次赋值, 赋值完成后不会继续监听该值的变化. 对于特定的场景是非常好用的指令

<template>
    <div>
        <p>hello {{world}}</p>
        <p v-text="'hello ' + world"></p>
        <p v-once="`hello ${world}`"></p>
        <button @click="world='vue2.0'">改变wrold值</button>
     </div>
</template>

<script>

export default {
     data () {
         return {
              world : "world"
         }
     }
}

</script>

同样通过上一个demo,稍作修改就可以体验下该指令;

点击按钮的时候 通过{{}} 和 v-text 绑定的 world 会改变,而 v-once 绑定则保持不变;

3.绑定HTML原文

有些特定场景需要 绑定html原文, 而{{}}和 v-text 两种绑定方法会把原文当作文本解析.

所以 vue 提供了另一个指令 v-html

同样用前面的demo稍作修改:

<template>
    <div>
        <p v-text="html"></p>
        <p v-html="html"></p>
     </div>
</template>

<script>

export default {
     data () {
         return {
              html : `<span style="color:blue">我是蓝色的</span>`
         }
     }
}

</script>

我们可以清楚的看到通过v-text绑定的html, 直接把标签当作文本解析了,

通过v-html绑定的html, 则把html正常解析了, 文字也显示为蓝色.

4.属性绑定

vue中非常重要的一个用法,组件中传递!对于属性的绑定我们不能使用{{attr}}语法,同样vue提供给了属性绑定指令

v-bind:attr="attr" 对应语法糖 :attr="attr"

简单的应用场景:

<template>
    <div>
    	<span v-bind:id="id">id</span>
    	<a :href="href">vue</a>
    	<img :src="img" alt="image">
    </div>
</template>

<script>

export default {
     data () {
         return {
              id: 'vue',
              href: 'https://cn.vuejs.org',
              img: 'https://cn.vuejs.org/images/logo.png'
         }
     }
}

</script>

属性中支持number string boolean类型，不用:来绑定的属性可以直接属性赋值(换句话说,如果不是需要动态赋值的属

性,可以直接按照html语法的属性写)，如果需要动态改变属性值(通过data数据中返回的属性变量)一定要使用:

5.使用JavaScript表达式

vue 绑定语法{{}}支持JavaScript表达式(简单的运算,不支持流程控制的语法)

<template>
     <div>
        <div id="method1">
            <p>{{ count < 0 ? '+' : '-'}} {{count + 1}}</p>
            <button @click='count ++'>增加</button>
            <button @click='count --'>减少</button>
        </div>
        <div id="method2">
            <input type="text" v-model='message'>
            <h1>{{ message.split('').reverse().join('') }}</h1>
        </div>
     </div>
</template>

<script>

export default {
     data () {
         return {
              count : 0,
              message : ""
         }
     }
}

</script>

使用中,如果是特别复杂的算法,建议通过method或者filter来计算;

6.修饰符

修饰符（Modifiers）是以半角句号 . 指明的特殊后缀，用于指出一个指令应该以特殊方式绑定.

对于vue绑定的事件, 可以提前通过修饰符限制事件方法执行的过程

例如: 通过 click.prevent 来阻止默认行为

<template>
     <div>
         <a href="https://www.baidu.com" @click.prevent="click">baidu</my-button>
     </div>
</template>

<script>
export default {
     methods : {
         click () {
             alert("阻止a标签的默认行为 .prevent")
         }
     }
}

</script>

再有: 通过 v-model.trim 来去掉value的空格

<template>
     <div>
        <input type="text" v-model.trim="content">
        <input type="text" v-model = 'content'>
     </div>
</template>

<script>
export default {
     data () {
         return {
             content : ""
         }
     }
}
</script>

7.过滤器

Vue.js 允许你自定义过滤器，可被用作一些常见的文本格式化。过滤器可以用在两个地方：mustache 插值和 v-bind 表达

式。过滤器应该被添加在 JavaScript 表达式的尾部，由“管道”符指示：

在v-bind中也就是2.10的时候才支持这个功能，这个大家要对这个版本有一定的了解，正确的使用

应用场景 ： 把 账单金额格式化为$100,000.00

 <template>
     <div>
         <p>{{money | currency}}</p>
     </div>
</template>

<script>
export default {
     data () {
         return {
             telephone : 10000
         }
     },
     filter: {
     	currency (value) {
  		let money = value * 1
  		if( !!isNaN(money)) money = 0;
  		money.toLocaleString('en-IN', {
  			style: 'currency',
  			currency: 'CNY'
		});
		return money
	   }
     }
}
</script>
